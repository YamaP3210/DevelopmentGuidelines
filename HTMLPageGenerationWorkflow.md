# HTML Page Generation Workflow

この文書は、公開用 HTML ページを新規作成または大幅更新するときの作業手順を定める。
目的は、公開本文と内部情報を工程で分離し、作業メモや AI 向け制約が成果物へ混入することを防ぐことにある。

## 適用範囲

- 公開用 HTML ページの新規作成
- 公開用 HTML ページの全面改稿
- 外部標準を基にした規約ページの作成

## 原則

- 公開 HTML には、読者向け本文だけを書く。
- AI 向け方針、作業手順、進捗説明、退避理由、構築事情は公開 HTML に書かない。
- 標準規約、独自規約、内部メモは同じ段階で混在させない。
- 最終 HTML を書く段階では、公開可能な情報だけを使う。

## 手順

### 1. 取得

- 公式ソース、既存公開資料、必要なローカル文書を収集する。
- この段階では HTML を書かない。
- 出力は、取得した事実の一覧または規約項目一覧だけにする。

### 2. 分類

- 取得した情報を次の 3 区分に分ける。
- `公開標準情報`
- `公開独自情報`
- `内部情報`
- この段階でも HTML は書かない。
- 出力は、区分済みの一覧またはアウトラインにする。

### 3. 公開用入力の固定

- これから作るページが標準ページなら `公開標準情報` だけを確定版として残す。
- 独自規約ページなら `公開独自情報` だけを確定版として残す。
- `内部情報` はこの時点で最終 HTML の入力対象から外す。
- 可能なら、公開用入力を別ファイルに切り出してから次へ進む。

### 4. HTML 生成

- 最終 HTML は、前段で固定した公開用入力だけを基に書く。
- 説明は読者向けに書く。
- `このページは再構成した`、`退避した`、`ベースとして扱う`、`AI`、`作業方針` のような内部文脈語を本文に入れない。
- 概要欄には、読者がそのページで何を得られるかだけを書く。

### 5. 検査

- 生成後に、公開 HTML に内部情報が混ざっていないか確認する。
- 少なくとも次の観点を確認する。
- 読者向け本文だけになっているか。
- 作業経緯や進捗説明が入っていないか。
- AI、作業、退避、再構成、ベース、取得範囲、方針、内部、進捗、検証のような内部運用語が本文に残っていないか。
- 公式ソースや確定済み公開入力に含まれない説明が紛れ込んでいないか。

## 完了条件

- 公開 HTML に内部情報がない。
- ページ本文が読者向けの規約説明だけで構成されている。
- 参照した公開入力の内容が HTML に反映されている。
- 標準規約と独自規約が混在していない。

## 禁止

- 情報取得と最終 HTML 生成を同時に進めること。
- 内部メモを見たまま、その内容を濾過せず HTML 本文へ書くこと。
- 公開ページの概要欄に、構築事情や運用方針を書くこと。
- 未分類の情報をそのまま本文へ流し込むこと。
