# HTML Page Generation Workflow

この文書は、公開用 HTML ページを新規作成または大幅更新するときの作業手順を定める。
目的は、公開本文と内部情報を工程で分離し、作業メモや AI 向け制約が成果物へ混入することを防ぐことにある。

## 適用範囲

- 公開用 HTML ページの新規作成
- 公開用 HTML ページの全面改稿
- 外部標準を基にした規約ページの作成

## 原則

- 公開 HTML には、読者向け本文だけを書く。
- 最終 HTML の入力には、公開可能な情報だけを使う。
- 標準規約、独自規約、内部情報は区分したうえで扱い、未分類のまま混在させない。

## 手順

### 1. 取得

- 公式ソース、既存公開資料、必要なローカル文書を収集する。
- この段階では、公開用 HTML 本文をまだ書かない。
- 出力は、取得した事実の一覧または規約項目一覧に留める。

### 2. 分類

- 取得した情報を次の 3 区分に分ける。
- `公開標準情報`
- `公開独自情報`
- `内部情報`
- 出力は、区分済みの一覧またはアウトラインにする。

### 3. 公開用入力の固定

- これから作るページが標準ページなら `公開標準情報` だけを確定版として残す。
- 独自規約ページなら `公開独自情報` だけを確定版として残す。
- `内部情報` はこの時点で最終 HTML の入力対象から外す。
- 最終 HTML の入力に使う情報は、この段階で明示的に確定してから次へ進む。

### 4. HTML 生成

- 最終 HTML は、前段で固定した公開用入力だけを基に書く。
- 説明は読者向けに書く。
- 概要を含む本文全体は、読者がそのページで何を得られるかが分かる内容だけで構成する。

### 5. 検査

- 生成後に、公開 HTML に内部情報が混ざっていないか確認する。
- 少なくとも次の観点を確認する。
- 読者向け本文だけになっているか。
- 作業経緯や進捗説明が入っていないか。
- 確定した公開用入力に含まれない内部情報や補助説明が紛れ込んでいないか。

## 完了条件

- 公開 HTML に内部情報がない。
- ページ本文が読者向けの規約説明だけで構成されている。
- 確定した公開用入力の内容が HTML に反映されている。

## 禁止

- 情報取得と最終 HTML 生成を同時に進めること。
- 内部メモを見たまま、その内容を濾過せず HTML 本文へ書くこと。
- 未分類の情報をそのまま本文へ流し込むこと。
