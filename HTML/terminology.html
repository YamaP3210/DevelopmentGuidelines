概要

システム開発における様々な用語・呼称を統一し、解釈のブレをなくします。

開発単位

このガイドラインでは、開発単位を表す語として ケース を使います。ソリューション と プロジェクト には、開発単位そのものを指す意味を含めません。

用語 定義
ケース システム開発における 1 つの開発単位を表す語です。機能、案件、製品、モジュールなどをまとめて扱う単位として使います。一般的に「〇〇開発プロジェクト」と呼ぶ場面でも、このガイドラインでは「〇〇開発ケース」と呼びます。
ソリューション Visual Studio などの IDE において、複数のプロジェクトをまとめて管理する構成単位を表す語です。このガイドラインでは、開発単位そのものを指す意味では使いません。
プロジェクト Visual Studio などの IDE において、ビルドや参照設定の単位となる構成要素を表す語です。このガイドラインでは、開発単位そのものを指す意味では使いません。

一般的な文脈では、ソリューション や プロジェクト が開発単位そのものを表す場合がありますが、 このガイドラインではその意味を採用しません。開発単位を表す場合は、常に ケース を使います。
