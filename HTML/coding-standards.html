概要

ここでは、C# のコーディング規約を、命名、言語スタイル、書式、適用方法に分けて整理しています。このコーディング規約は、.NET / C# の標準規約をベースにカスタマイズした内容となっています。

共通ガイドライン

命名や formatter の設定値だけでは表せない、C# の書き方に関する基本的な考え方をまとめています。

言語機能の使い方

  • 可能な限り新しい C# 言語機能と現行の書き方を使います。
  • 古い構文や回りくどい書き方は避けます。
  • 読みやすさと単純さを優先し、不要に複雑なコードを避けます。
  • I/O バウンド処理では async / await を使います。
  • コレクション処理では LINQ を読みやすさの範囲で活用します。
  • 型名は string や int などの C# キーワードを優先します。
  • var は式から型が明確に読み取れる場合に使います。
  • 符号なし型より int を優先します。

文字列・初期化・コレクション

  • 短い文字列連結は文字列補間を使います。
  • ループ内で大量の文字列を連結するときは StringBuilder を使います。
  • エスケープや verbatim 文字列より raw string literal を優先します。
  • オブジェクト生成では簡潔な new() やオブジェクト初期化子を使います。
  • コレクション初期化では collection expressions を使います。
  • 必須初期化が必要な値には required を検討します。

例外・制御・イベント

  • 捕捉するのは適切に処理できる例外だけにします。
  • using で表現できる解放処理に、解放目的だけの try-finally は使いません。
  • 比較では && と || を優先します。
  • 単純な delegate は Func<> と Action<> を優先します。
  • 後で解除しないイベント ハンドラーは lambda で書けます。

命名

命名は、同じ種類の要素が同じ規則で読めることと、名前を見ただけで役割が分かることを優先します。

対象ごとの命名規約

対象 規約
class / struct / enum ケース識別子 + PascalCase
interface I + ケース識別子 + PascalCase
namespace ケース識別子をルートにした階層名
public / protected 系メンバー PascalCase
property PascalCase
private / internal field _camelCase
private / internal static field s_ + camelCase
thread static field t_ + camelCase
method parameter / local variable camelCase
const PascalCase

命名方針

  • 名前空間・クラス名・設定名・資産名は、ケース識別子を先頭に付けた名前で統一します。
  • 名前空間のルートはケース識別子から始め、XXX.Client.UI、XXX.Client.Tools、XXX.Server.Core のように役割ごとに分けます。
  • ケース間で名前空間を共有しません。
  • 名前は短さより明確さを優先し、意味が伝わるものにします。
  • 略語や頭文字語は、一般に通じるものを使い、ID、TCP、UI のように大文字のまま維持します。
  • DataBase を DB とするような、意味が通る略称は許容します。
  • attribute 型は Attribute で終える名前にします。
  • enum は通常は単数形、flags enum は複数形にします。
  • 単文字名は避け、ループ変数にも意味が分かる名前を使います。
  • 必要な場合は @if のような verbatim identifier を使えます。
  • 連続する __ は使いません。

言語・不要コード規約

C# の書き方や不要コードの扱いについて、日常的に判断が必要になる規約をまとめています。

波括弧

  • if、for、foreach、while の本体は、1 行でも波括弧を付けます。
if (isReady)
{
    Start();
}

foreach (var item in items)
{
    Process(item);
}

型名と var

  • 組み込み型は string、int、bool などの C# キーワード型を使います。
  • var は、右辺を見れば型が明確に分かる場合だけ使います。
string title = "Guidelines";
int retryCount = 3;

var user = new UserProfile();
var values = new List<int>();

UserProfile profile = CreateProfile();
RequestContext context = BuildContext();

using と解放処理

  • 解放が必要なオブジェクトは using で扱い、解放目的だけの try-finally は使いません。
  • 不要な using ディレクティブは残しません。
using var stream = File.OpenRead(path);
using var reader = new StreamReader(stream);

var content = reader.ReadToEnd();

条件式と null 判定

  • null 判定や条件式は、読みやすい書き方を優先して簡潔に書きます。
  • 不要な比較や不要な null 許容演算子は残しません。
if (user is null)
{
    return;
}

var displayName = user.Profile?.DisplayName ?? "Unknown";

switch とパターンマッチ

  • 条件分岐を式として自然に書ける場合は、switch 式やパターンマッチを使います。
  • 型判定とキャストを続けて書く場合は、パターンマッチでまとめます。
var message = status switch
{
    Status.Success => "Completed",
    Status.Failed => "Failed",
    _ => "Pending"
};

if (request is LoginRequest loginRequest)
{
    Validate(loginRequest);
}

readonly と不要コード

  • 再代入しないフィールドには readonly を付けます。
  • 不要な引数、不要な suppression、不要なコードは残しません。
private readonly ILogger _logger;
private readonly IRepository _repository;

public UserService(ILogger logger, IRepository repository)
{
    _logger = logger;
    _repository = repository;
}

書式規約

改行、字下げ、空白、空行の入れ方について、実際のコードでどう書くかをまとめています。

改行と波括弧

  • 波括弧は新しい行に置きます。
  • else、catch、finally は新しい行に置きます。
  • switch の case 本文はインデントし、ラベル自体もインデントします。
if (isValid)
{
    Execute();
}
else
{
    Recover();
}

switch (status)
{
    case Status.Ready:
        Start();
        break;
}

using の並び

  • using はカテゴリごとに分け、カテゴリ間を 1 行空けます。
using System;
using System.Collections.Generic;

using Cysharp.Threading.Tasks;

using XXX.Client.UI;
using XXX.Server.Core;

空行の入れ方

  • namespace の前を 5 行空けます。
  • クラス内部ではカテゴリ切り替えを 5 行空けます。
  • 同一カテゴリ内の項目は 3 行空けます。
  • メソッド内の処理の塊は 1 行空けます。
using System;
using System.Collections.Generic;





namespace XXX.Client.UI
{
    public sealed class XXXUserPanel
    {
        private readonly ILogger _logger;



        private readonly IRepository _repository;





        public void Initialize()
        {
            LoadSettings();

            BindEvents();
        }
    }
}

空白

  • キーワードの後、カンマの後、二項演算子の前後には空白を入れます。
  • メソッド名と開き括弧の間、キャストの直後、ドットの前後には不要な空白を入れません。
if (isEnabled && retryCount > 0)
{
    Save(userId, retryCount);
}

var value = (int)count;
result = service.Execute(request);

1 行保持

  • 1 行で書ける単純な文でも、可読性を損なう場合は複数行に展開します。
  • 複数の処理を 1 行に詰め込まず、読みやすさを優先します。
if (shouldNotify)
{
    Notify();
}

return new UserSummary
{
    Id = user.Id,
    Name = user.Name
};

サンプルコード

ここでは、命名、書式、言語機能の使い方をまとめて確認できるように、サンプルクラスのコードを掲載しています。

開く ▲閉じる 
using System;
using System.Collections.Generic;
using System.IO;
using System.Linq;





namespace XXX.Client.UI
{
    public sealed class XXXClassName
    {
        private static readonly string s_defaultTitle = "XXX Default Title";



        private readonly IXXXRepository _repository;



        private readonly ILogger _logger;





        private int _retryCount;



        public string Title { get; private set; }



        public IReadOnlyList<string> Items { get; private set; }



        public XXXClassName(IXXXRepository repository, ILogger logger)
        {
            _repository = repository;
            _logger = logger;
            Title = s_defaultTitle;
            Items = Array.Empty<string>();
        }





        public void Initialize(IEnumerable<string> sourceItems)
        {
            ArgumentNullException.ThrowIfNull(sourceItems);

            Items = sourceItems
                .Where(item => !string.IsNullOrWhiteSpace(item))
                .Select(item => item.Trim())
                .ToArray();

            Title = BuildTitle(Items.Count);
        }



        public string Execute(bool isEnabled, string? basePath)
        {
            if (!isEnabled)
            {
                return "Disabled";
            }

            if (basePath is null)
            {
                return "Path is required";
            }

            using var stream = File.OpenRead(basePath);
            using var reader = new StreamReader(stream);

            var content = reader.ReadToEnd();
            var summary = content.Length switch
            {
                0 => "Empty",
                < 100 => "Short",
                < 500 => "Medium",
                _ => "Long"
            };

            for (var itemIndex = 0; itemIndex < Items.Count; itemIndex++)
            {
                var itemName = Items[itemIndex];

                if (string.IsNullOrWhiteSpace(itemName))
                {
                    continue;
                }

                _logger.Log($"Processing: {itemName}");
            }

            foreach (var itemName in Items)
            {
                if (itemName.Length < 3)
                {
                    continue;
                }

                _repository.Save(itemName, summary);
            }

            while (_retryCount < 3)
            {
                if (TryComplete())
                {
                    break;
                }

                _retryCount++;
            }

            return summary;
        }



        private string BuildTitle(int itemCount)
        {
            var label = itemCount switch
            {
                0 => "No Items",
                1 => "Single Item",
                _ => $"{itemCount} Items"
            };

            return $"XXX {label}";
        }



        private bool TryComplete()
        {
            var timestamp = DateTime.UtcNow;

            _logger.Log($"Retry Count: {_retryCount}, Timestamp: {timestamp:O}");

            return _retryCount >= 1;
        }
    }





    public interface IXXXRepository
    {
        void Save(string itemName, string summary);
    }





    public interface ILogger
    {
        void Log(string message);
    }
}
