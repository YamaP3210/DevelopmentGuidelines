概要

ここでは、Microsoft Learn で公開されている .NET / C# の標準規約を、命名、言語スタイル、書式、適用方法に分けて整理しています。

C# のコードを書くときに、何を基準に名前を付け、どのような書式や style rule を採用するかを確認するための参照ページとして使えます。

共通ガイドライン

公式の文章ベース規約は、命名や formatter 設定だけでなく、サンプルや通常コードでどう書くかの判断基準も含みます。

言語機能の使い方

  • 可能な限り新しい C# 言語機能と現行の書き方を使います。
  • 古い構文や回りくどい書き方は避けます。
  • 読みやすさと単純さを優先し、不要に複雑なコードを避けます。
  • I/O バウンド処理では async / await を使います。
  • コレクション処理では LINQ を読みやすさの範囲で活用します。
  • 型名は string や int などの C# キーワードを優先します。
  • var は式から型が明確に読み取れる場合に使います。
  • 符号なし型より int を優先します。

文字列・初期化・コレクション

  • 短い文字列連結は文字列補間を使います。
  • ループ内で大量の文字列を連結するときは StringBuilder を使います。
  • エスケープや verbatim 文字列より raw string literal を優先します。
  • オブジェクト生成では簡潔な new() やオブジェクト初期化子を使います。
  • コレクション初期化では collection expressions を使います。
  • 必須初期化が必要な値には required を検討します。

例外・制御・イベント

  • 捕捉するのは適切に処理できる例外だけにします。
  • using で表現できる解放処理に、解放目的だけの try-finally は使いません。
  • 比較では && と || を優先します。
  • 単純な delegate は Func<> と Action<> を優先します。
  • 後で解除しないイベント ハンドラーは lambda で書けます。

命名

命名は、識別子として合法かどうかの規則と、.NET が既定で推奨する慣例の 2 層で扱います。

識別子の基本規則

  • 識別子は文字または _ で開始します。
  • 識別子には Unicode 文字、数字、接続文字、結合文字、書式文字を含められます。
  • キーワードを識別子として使う必要がある場合は @if のように verbatim identifier を使います。
  • 連続する __ は使いません。

既定の命名慣例

対象 標準的な慣例
class / struct / enum / delegate / namespace PascalCase
interface I 接頭辞 + PascalCase
public / protected 系メンバー PascalCase
private / internal field _camelCase
private / internal static field s_ 接頭辞 + camelCase
thread static field t_ 接頭辞 + camelCase
method parameter / local variable camelCase
const PascalCase
record の primary constructor parameter PascalCase
class / struct の primary constructor parameter camelCase

標準的な命名の考え方

  • 名前は短さより明確さを優先し、意味が伝わるものにします。
  • 略語や頭字語は、広く定着したものに限って使います。
  • 属性型は Attribute で終える命名にします。
  • enum は通常は単数形、flags enum は複数形を使います。
  • 名前空間は用途が分かる名前にし、reverse domain name notation に沿って構成します。
  • アセンブリ名は、そのアセンブリの主目的が分かる名前にします。
  • 単文字名は、単純なループカウンターなどの限定的な場面を除いて避けます。

editorconfig で表現する命名ルール

公式の naming rules は、symbol group、naming style、naming rule の 3 要素で組み立てます。既定ルールの違反は IDE1006 として扱われます。

  • 命名スタイルでは capitalization、required_prefix、required_suffix、word_separator を定義できます。
  • symbol group では applicable_kinds、applicable_accessibilities、required_modifiers を定義できます。
  • build 時に厳格に扱う場合は、命名ルール側だけでなく dotnet_diagnostic.IDE1006.severity も設定します。

書式で特に重要な標準既定

  • 波括弧位置の既定は csharp_new_line_before_open_brace = all です。つまり Allman スタイルです。
  • else、catch、finally は新しい行に置くのが既定です。
  • switch の case 本文はインデントし、ラベル自体もインデントするのが既定です。

適用方法

公式規約は、文章の推奨だけでなく analyzer と formatter で適用する前提です。 実装では .editorconfig を基準にし、必要なものは dotnet_diagnostic.<rule id>.severity で build 時の扱いを決めます。

  • 書式は IDE0055 と formatting option 群で管理します。
  • 命名は IDE1006 と naming rule 群で管理します。
  • 言語スタイルは IDE000x から IDE03xx までの style rules で管理します。
  • Visual Studio の個人設定だけに依存せず、リポジトリ内の .editorconfig を優先します。
