C# コーディング規約

ここには C# の実装規約を集約します。項目ごとに細分化して追記していきます。

分類 記録内容の例
命名 型名、メンバー名、ローカル変数、イベント、インターフェースの付け方
構文 var、record、using、パターンマッチ、初期化子
例外 再送出、境界での捕捉、業務例外とシステム例外の分離
非同期 async / await、キャンセル、同期ブロックの回避
データ表現 DTO、Entity、値オブジェクト、列挙型、コレクションの扱い

必要に応じて、ここに詳細ルールの章を追加します。

基本アーキテクチャ

この章では、開発時に採用する構造を整理します。MVC、MVVM、レイヤード構成などの前提をここに書きます。

考える観点

  • 責務分割が明確か
  • 依存方向が一方通行か
  • UI、状態、ドメイン、I/O が混ざっていないか

記録する内容

  • MVC / MVVM の使い分け
  • Presenter や Service の位置付け
  • Unity でのシーン構成や責務の割り当て

Visual Studio セッティング

Visual Studio 側で固定したい設定や拡張機能を整理します。

  • コードフォーマットとインデントの標準
  • .editorconfig と分析ルールの運用
  • 必要な拡張機能、テンプレート、スニペット
  • デバッグ、プロファイル、ショートカットの共通化
  • ソリューション構成、起動プロジェクト、ビルド構成

テスト方針

テストは、規約を実装に定着させるための手段として扱います。

  • ロジックの中心はユニットテストで担保する
  • Unity 特有の挙動は必要に応じて PlayMode テストで確認する
  • バグ修正は再発防止テストを優先する
  • テスト名は前提、操作、期待結果が読める形にする

レビュー観点

  • 命名が意味を表しているか
  • 責務が適切に分かれているか
  • 例外、null、非同期、ログの扱いに無理がないか
  • 設定や手順が環境差分に耐えられるか
  • 仕様変更時に関連文書も更新されるか
