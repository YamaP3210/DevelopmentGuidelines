C# 独自上書き規約

このページは、.NET 標準には含まれないが、このプロジェクトで独自に定義していた C# 規約を退避したものです。 ベース規約と混在させず、必要な上書きだけを明示的に参照できるようにしています。

区分: プロジェクト独自 用途: 上書き候補の退避

目次

  • 標準ベースへ戻る
  • 命名上書き
  • 書式上書き
  • 扱い方

命名上書き

  • ケース識別子を名前空間・クラス名・設定名・資産名の接頭辞として使う。
  • 名前空間のルートをケース識別子から始め、XXX.Client.UI、XXX.Client.Tools、XXX.Server.Core のように分割する。
  • ケース間で名前空間を共有しない。
  • 略語・頭文字語は通常の PascalCase に丸めず、ID、TCP、UI のように大文字のまま維持する。
  • 単文字名を避け、ループ変数にも意味が分かる名前を優先する。
  • DataBase を DB とするような、意味が通る略称を許容する。

書式上書き

  • using をカテゴリごとに分け、カテゴリ間を 1 行空ける。
  • namespace の前を 5 行空ける。
  • クラス内部ではカテゴリ切り替えを 5 行空ける。
  • 同一カテゴリ内の項目は 3 行空ける。
  • メソッド内の処理の塊は 1 行空ける。

扱い方

これらは .NET 標準ではありません。採用する場合は、標準規約を上書きする理由を明示したうえで、別段のプロジェクト規約として扱ってください。

元の混在状態を避けるため、標準ベースの コーディング規約 / C# から分離しています。
