C# 独自規約

このページは、C# の標準規約に加えて、このプロジェクトで採用する独自規約を整理したページです。

区分: プロジェクト独自 用途: 追加規約

目次

  • 標準規約へ戻る
  • 命名規約
  • 書式規約
  • 扱い方

命名規約

  • ケース識別子を名前空間・クラス名・設定名・資産名の接頭辞として使う。
  • 名前空間のルートをケース識別子から始め、XXX.Client.UI、XXX.Client.Tools、XXX.Server.Core のように分割する。
  • ケース間で名前空間を共有しない。
  • 略語・頭文字語は通常の PascalCase に丸めず、ID、TCP、UI のように大文字のまま維持する。
  • 単文字名を避け、ループ変数にも意味が分かる名前を優先する。
  • DataBase を DB とするような、意味が通る略称を許容する。

書式規約

  • using をカテゴリごとに分け、カテゴリ間を 1 行空ける。
  • namespace の前を 5 行空ける。
  • クラス内部ではカテゴリ切り替えを 5 行空ける。
  • 同一カテゴリ内の項目は 3 行空ける。
  • メソッド内の処理の塊は 1 行空ける。

扱い方

これらは、標準規約に追加してこのプロジェクトで採用する規約です。C# の実装では、標準規約とあわせて参照してください。

標準的な規約は コーディング規約 / C# を参照してください。
